# 50days-Coding-Challenge

**Day-1**

**Excel Data Analyst Challenge – Sales Data Analysis**

📌 **Project Overview**

This project is part of a Data Analyst practice challenge where sales data is analyzed using Microsoft Excel.
The objective of the challenge is to perform data exploration, calculate revenue, create pivot tables, and derive meaningful business insights from the dataset.
The dataset contains 200 sales transactions including product details, regions, quantities, and prices.

📂 **Dataset**

The dataset contains the following fields:

Column    -->  	Description

Order_ID  -->	Unique order identifier

Order_Date  --> Date of the order

Region	Sales region  --> (North, South, East, West)

Product	-->  Name of the product

Category  -->  Product category

Quantity  -->  Number of units sold

Unit_Price  -->  Price per unit

🧹 **Data Cleaning**

Before performing the analysis, the dataset was reviewed to ensure it was clean and ready for analysis.

Steps performed:
•Verified column data types

•Checked for missing values

•Ensured numeric formatting for quantity and price

•Created a calculated column for Revenue

📊 **Data Analysis**

Using Pivot Tables in Excel, the dataset was analyzed to answer key business questions.

Analysis Performed

•Total Revenue by Region

•Revenue by Product Category

•Top Performing Products

•Sales distribution across regions

📈 **Visualizations**

Charts were created in Excel to better understand the sales performance.

Visualizations include:

•Revenue by Region (Column Chart)

•Category-wise Revenue Comparison

•Product Performance Analysis

🔎 **Key Insights**

•West region generated the highest revenue.

•Electronics category contributed the majority of sales.

•Desk Chair and Notebook are among the top-performing products.

•North region recorded the lowest revenue, indicating a potential area for improvement.
