# 🖥️ IT Help Desk Ticket Analysis

📌**Project Overview**
This project focuses on analyzing IT help desk ticket data to identify patterns, improve operational efficiency, and provide actionable recommendations. The analysis helps uncover recurring issues, evaluate analyst performance, and optimize resource allocation.

🎯 **Objectives**

•	Analyze ticket volume trends over time

•	Evaluate resolution time across categories and priorities

•	Identify recurring issues and root causes

•	Assess analyst performance

•	Provide data-driven recommendations for IT improvement

## 📂 Dataset Description

The dataset contains 150 help desk tickets with the following fields:

•	ticket_id – Unique ticket identifier

•	submission_date – Ticket creation date and time

•	resolution_date – Ticket resolution date and time

•	category – Issue type (Network, Software, Hardware, Access)

•	assigned_analyst – IT staff handling the ticket

•	description – Problem description

•	priority – Ticket urgency (Low to Critical)

🛠️ **Tools & Technologies**

•	Microsoft Excel

•	Pivot Tables & Charts

•	Excel Formulas (IF, COUNTIF, AVERAGEIFS, TEXT, HOUR)

🔍**Key Insights**

•	Ticket volume peaks during specific days and working hours

•	Hardware issues have longer resolution times

•	Password and login issues are the most frequent problems

•	Significant variation exists in analyst performance

•	Many issues are repetitive and can be automated

🚀 **Recommendations**

•	Implement self-service password reset system

•	Improve network monitoring and infrastructure

•	Provide training for analysts with higher resolution times

•	Increase staffing during peak periods

•	Develop a knowledge base for common issues

•	Automate repetitive IT requests



📈 **Outcome**
This analysis enables the IT department to shift from reactive problem-solving to proactive management, improving efficiency, reducing workload, and enhancing user satisfaction.
